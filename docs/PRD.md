# PRD: 英语学习助手

## 1. 产品概述

### 1.1 背景

用户拥有完整课程素材，共 3,654 个文件，涵盖 6 个级别、72 周课程。素材已按级别→周→天→编号的层级整理好，但目前需要在文件管理器中逐个点击打开文件，使用体验差。

### 1.2 目标

构建一个本地 Web 应用，将课程素材以结构化、可交互的方式展示，实现：
- 双击启动，浏览器直接使用
- 按级别→周→天导航，自动顺序播放当天内容
- 支持 iPad 通过局域网访问
- 记录学习进度和打卡统计

### 1.3 目标用户

| 角色 | 描述 |
|------|------|
| 主要操作者 | 家长（负责选课、播放、打卡） |
| 主要观看者 | 小朋友（跟着听/看/读） |

### 1.4 核心价值

消除"在文件夹中逐个找文件点击打开"的痛点，一个入口看所有教学内容。

## 2. 素材分析

### 2.1 文件统计

| 类型 | 数量 | 用途 |
|------|------|------|
| mp3 | 963 | 音频：讲解、歌曲、朗读 |
| png | 669 | 图片：讲义、手卡、打卡卡 |
| mp4 | 620 | 视频：唱跳、手偶、场景演绎 |
| wav | 512 | 音频：高音质版（3-4阶） |
| jpg | 432 | 图片：讲义、书单 |
| pdf | 415 | 绘本扫描、手工材料、书单 |
| docx | 32 | 歌词、双语译文 |
| mov | 5 | 视频：手偶动画（3阶） |
| pptx | 2 | 精读绘本讲解 |

### 2.2 目录结构

```
english-study/
├── 预备级/          (12周, 5天/周)
├── 基础级/          (24周, 5天/周)
├── 陪伴营 1阶/      (12周+试听周, 7天/周)
├── 陪伴营 2阶/      (12周+试听周, 7天/周)
├── 陪伴营 3阶/      (12周, 7天/周)
└── 陪伴营 4阶/      (12周, 7天/周)
```

### 2.3 每日课程结构

陪伴营 1-2阶（7天）：
```
Day1-儿歌听唱      → 讲义图 → 讲解音频 → 儿歌音视频 → 打卡
Day2-日常对话      → 讲义图 → 讲解音频 → 对话音视频 → 打卡
Day3-绘本讲读      → 讲义图 → 讲解音频 → 绘本PDF → 打卡
Day4-绘本阅读      → 讲义图 → 讲解音频 → 绘本PDF → 打卡
Day5-游戏+泛读     → 讲义图 → 讲解音频 → 游戏视频 → 打卡
Day6-分级读物      → 讲义图 → 讲解音频 → 分级PDF → 打卡
Day7-晓华讲堂      → 讲义图 → 讲解音频 → 打卡
```

预备级/基础级（5天）：
```
第一天-主题儿歌    → 讲义图 → 讲解音频 → 儿歌音视频 → 打卡
第二天-主题故事    → 讲义图 → 讲解音频 → 故事音视频 → 打卡
第三天-主题拓展    → 讲义图 → 讲解音频 → 打卡
第四天-亲子互动    → 讲义图 → 讲解音频 → 打卡
第五天-复习+家长课堂 → 讲义图 → 讲解音频 → 打卡
```

### 2.4 文件命名规范

文件按编号排序：`01-`, `02-`, `03-`, ...，编号决定展示和播放顺序。

## 3. 功能需求

### 3.1 启动与部署

| ID | 需求 | 优先级 |
|----|------|--------|
| F-01 | 双击 `start.command` 启动，自动扫描目录 + 启动服务器 + 打开浏览器 | P0 |
| F-02 | 启动时自动扫描素材目录生成 `data.json` 索引文件 | P0 |
| F-03 | 使用 Python3 内置 HTTP 服务器，零外部依赖 | P0 |
| F-04 | iPad 通过 `http://<电脑IP>:8080` 访问 | P0 |

### 3.2 导航与浏览

| ID | 需求 | 优先级 |
|----|------|--------|
| F-10 | 左侧导航栏：级别选择 → 周次列表 → 天次选择 | P0 |
| F-11 | 6 个级别可切换：预备级、基础级、陪伴营 1-4阶 | P0 |
| F-12 | 每个级别显示周次列表，显示周主题（英文书名） | P0 |
| F-13 | 每周显示天次列表，已打卡的天次标记 ✓ | P0 |
| F-14 | 首页默认显示上次学习的位置 | P1 |

### 3.3 内容展示

| ID | 需求 | 优先级 |
|----|------|--------|
| F-20 | 右侧内容区按编号顺序（01→06）展示当天所有文件 | P0 |
| F-21 | 每个编号项可折叠/展开，当前播放项自动展开 | P0 |
| F-22 | 图片（讲义/手卡）：展开后内嵌显示，支持多图轮播 | P0 |
| F-23 | 音频（mp3/wav）：展开后内嵌播放器，播完自动播放下一个 | P0 |
| F-24 | 视频（mp4/mov）：展开后内嵌播放器，支持 playsinline | P0 |
| F-25 | PDF（绘本）：展开后用 PDF.js 内嵌渲染，支持翻页和缩放 | P0 |
| F-26 | DOCX（歌词/译文）：构建时提取为 HTML，展开后内嵌显示 | P0 |
| F-27 | PPTX：暂不处理，标记为"请用其他应用查看" | P2 |

### 3.4 播放控制

| ID | 需求 | 优先级 |
|----|------|--------|
| F-30 | 混合模式：默认自动顺序播放，可手动跳转任意项 | P0 |
| F-31 | 点击"开始今天"按钮，从第一项开始自动播放 | P0 |
| F-32 | 音频/视频播放完毕后，自动展开并播放下一项 | P0 |
| F-33 | 手动点击任意项可跳转，打断自动播放序列 | P0 |
| F-34 | 图片项自动轮播，每张停留 5 秒 | P1 |
| F-35 | PDF 项手动翻页，不自动推进 | P1 |

### 3.5 进度与打卡

| ID | 需求 | 优先级 |
|----|------|--------|
| F-40 | 每天内容底部显示"✓ 今日完成"按钮 | P0 |
| F-41 | 点击打卡后，左侧天次列表标记 ✓ | P0 |
| F-42 | 首页顶部显示学习统计：已学天数、连续学习天数、完成率 | P0 |
| F-43 | 进度数据存储在 localStorage | P0 |
| F-44 | 记录上次学习的级别/周/天，下次打开自动恢复 | P1 |

| ID | 需求 | 优先级 |

| F-51 | 平板端左侧导航可收起，内容区全屏 | P0 |
| F-52 | 视频播放器支持 iOS Safari 的 playsinline | P0 |

## 4. 非功能需求

| ID | 需求 | 说明 |
|----|------|------|
| NF-01 | 零外部运行时依赖 | 仅需 Python3（macOS 自带） |
| NF-02 | 启动时间 < 5 秒 | 扫描 3,654 个文件 + 启动服务器 |
| NF-03 | 页面加载 < 2 秒 | data.json 预生成，无需运行时扫描 |
| NF-04 | 素材目录不做任何修改 | 只读访问原始文件 |
| NF-05 | 离线可用 | 不依赖外部 CDN，PDF.js 离线内嵌 |

---

## 5. 不在范围内

- 用户登录/多用户支持
- 云端同步进度
- 素材编辑/管理功能
- 课程内容本身的修改
- 移动端原生应用（iOS/Android）
- 自动更新机制
